#### Flujo de trabajo con Pull Request
1. Crear una rama de tarea desde `dev`.
2. Implementar cambios y hacer commits claros.
3. Abrir **Pull Request** desde la rama de tarea hacia `dev`.
4. Revisar, corregir y aprobar el PR.
5. Hacer merge a `dev`.
6. Cuando `dev` esté validada por la profesora, hacer merge de `dev` a `main`.

> **Nota de entrega:** el código que se debe presentar para aprobación es el de la rama **`dev`**.
